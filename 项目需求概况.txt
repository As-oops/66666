# 项目名称：My AI Chatbot（Streamlit 智能对话系统）

## 项目概述
开发一套简洁高效的网页版智能对话Chatbot系统，使用 Python+Streamlit 框架，适配 Windows 本地运行，支持 DeepSeek 等多种模型接入。

## 产品功能需求

### 1. 核心对话交互模块
- **用户输入与回复交互**：支持用户文本输入，AI助手实时回复
- **多轮对话支持**：基于 session_state 维护对话历史，支持连续对话
- **对话状态反馈**：提供"Thinking..."加载动画，提升用户体验
- **错误提示**：当API Key未配置时，显示友好的错误提示信息

### 2. 模型配置与参数调节模块
- **侧边栏配置面板**：提供完整的配置界面
  - API Key 输入（支持密码隐藏）
  - Base URL 配置（默认 DeepSeek API）
  - 模型名称配置（默认 deepseek-chat）
  - 角色预设选择
  - 自定义系统提示词
  - 温度参数调节（0.0-1.5）
- **环境变量支持**：支持通过 .env 文件或系统环境变量配置
  - DEEPSEEK_API_KEY
  - OPENAI_API_KEY
  - LLM_BASE_URL
  - LLM_MODEL
- **Streamlit Secrets 支持**：支持通过 st.secrets 配置敏感信息

### 3. 角色预设模块
提供多种预设角色，用户可快速切换：
- **通用助手**：清晰、自然、直接的回答
- **Python老师**：严格但友好的 Python 教学，优先提示思路
- **英语陪练**：简单英语对话，附带中文解释
- **旅行规划师**：具体、实用、可执行的旅行建议
- **吐槽型朋友**：有趣但不冒犯的对话风格

### 4. 对话管理模块
- **清除对话功能**：一键清除当前对话历史
- **对话历史展示**：在主界面展示所有历史对话
- **角色切换**：切换角色时自动更新系统提示词

## 技术实现要求

### 1. 核心技术栈
- **编程语言**：Python 3.9+
- **Web框架**：Streamlit
- **API客户端**：OpenAI SDK（兼容多种模型）
- **配置管理**：python-dotenv（可选）

### 2. 代码结构要求
- **模块化设计**：
  - 配置获取函数（get_setting）
  - AI对话函数（ask_ai）
  - 主程序逻辑
- **会话状态管理**：使用 st.session_state 维护对话历史和配置状态
- **错误处理**：处理 API 调用异常和配置缺失情况

### 3. 用户界面要求
- **页面配置**：居中布局，页面标题"AI 星期八"，图标🤖
- **双栏布局**：
  - 侧边栏：配置面板
  - 主区域：对话界面
- **对话展示**：使用 st.chat_message 区分用户和助手消息
- **响应式设计**：适配不同屏幕尺寸
